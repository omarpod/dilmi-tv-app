# Dilmi TV — تطبيق الأندرويد (Flutter)

نسخة محدّثة بخمس ميزات جوهرية: رابط سيرفر ثابت (Render)، تعدد اللغات
(عربي/فرنسي)، الوضع المظلم/الفاتح، إشعارات محلية مُصلَحة، والمفضلة الذكية.

## 1. إكمال المشروع وتشغيله

```bash
flutter create .
flutter pub get
flutter pub run flutter_launcher_icons
flutter run
```

## 2. الرابط الثابت (Render) — لا حاجة لأي إعداد يدوي بعد الآن

`lib/config/api_config.dart` يحتوي رابط سيرفرك على Render مباشرة:
```dart
static const String productionBaseUrl = 'https://dilmi-tv-backend.onrender.com/api';
```
حذفنا بالكامل حقل إدخال IP اليدوي وزر "اختبار الاتصال" من شاشة الإعدادات
(لم تعد هناك حاجة لهما بعد استقرار السيرفر على رابط دائم).

**ملاحظة مهمة عن الخطة المجانية لـ Render:** السيرفر "ينام" تلقائياً بعد
فترة من عدم الاستخدام، وأول طلب بعد النوم قد يستغرق حتى 50 ثانية لإيقاظه.
جعلنا مهلة الطلب في `api_service.dart` أطول (60 ثانية) مع رسالة توضيحية
لهذا السبب تحديداً، حتى لا تظهر كخطأ حقيقي في الاتصال.

**إذا غيّرت السيرفر مستقبلاً** (نطاق خاص، استضافة أخرى...)، عدّل القيمة
أعلاه في مكان واحد فقط.

## 3. تعدد اللغات (عربي/فرنسي)

نظام ترجمة بسيط بدون تعقيد أدوات gen-l10n:
- `lib/l10n/app_strings.dart`: كل النصوص في مكان واحد، لكلا اللغتين.
- `lib/l10n/app_localizations.dart`: يوفر `context.tr('key')` لاستخدامه
  في أي شاشة.
- `lib/providers/locale_provider.dart`: يحفظ اختيار اللغة بشكل دائم.

**تبديل اللغة:** من شاشة الإعدادات ← قسم "اللغة". العربية تُفعّل اتجاه
RTL تلقائياً، والفرنسية LTR، دون أي كود إضافي (تتولى `flutter_localizations`
ذلك تلقائياً بناءً على `Locale` المختارة).

**لإضافة نص جديد لاحقاً:** أضف مفتاحاً جديداً في كلا القسمين (`ar` و `fr`)
داخل `app_strings.dart`، ثم استخدمه في أي شاشة بـ `context.tr('مفتاحك')`.

## 4. الوضع المظلم/الفاتح

- `lib/providers/theme_provider.dart`: يحفظ اختيار المستخدم بشكل دائم.
- `lib/config/app_theme.dart`: يحتوي الآن `darkThemeData` و `lightThemeData`
  كاملين، بالإضافة إلى `AppTextColors` (ألوان نصوص تتكيف تلقائياً حسب
  الوضع الحالي بدل الألوان الثابتة التي كانت مكتوبة يدوياً سابقاً).
- **التبديل:** من شاشة الإعدادات ← مفتاح "الوضع المظلم".
- شاشة الترحيب (Splash) تبقى داكنة دائماً بتصميم العلامة التجارية الثابت،
  بغض النظر عن اختيار المستخدم — وهذا مقصود ومطابق لتطبيقات مشابهة.

## 5. إصلاح الإشعارات

كانت المشكلة الأكثر شيوعاً: **عدم طلب صلاحية الإشعارات صراحة**. على
أندرويد 13 (API 33) فما فوق، لا تظهر أي إشعارات إطلاقاً دون هذه الخطوة،
دون أي خطأ ظاهر يشرح السبب. الإصلاحات في `notification_service.dart`:
1. طلب صلاحية `POST_NOTIFICATIONS` صراحة عبر `requestNotificationsPermission()`.
2. طلب صلاحيات iOS (alert/badge/sound) صراحة عند التهيئة.
3. إنشاء "قناة الإشعارات" (Notification Channel) صراحة بدل الاعتماد على
   إنشائها الضمني عند أول إشعار فقط.

جرّبها من شاشة الإعدادات ← "إرسال إشعار تجريبي".

**تذكير:** هذا يوفر آلية العرض فقط. لتفعيلها تلقائياً عند مباراة مباشرة
حقيقية، تحتاج لاحقاً فحصاً دورياً لـ `/api/matches/?status=live`، أو
الأفضل: ربط Django بـ Firebase Cloud Messaging لإشعارات Push حقيقية.

## 6. المفضلة الذكية (ميزة جديدة)

`lib/services/favorites_service.dart`: يحفظ معرّفات القنوات/الأخبار
المفضّلة محلياً عبر SharedPreferences، ويُبلّغ كل الشاشات المهتمة تلقائياً
عند أي تغيير (عبر `ChangeNotifier` + `Consumer`).

- من شاشتي **القنوات** و**الأخبار**: اضغط أيقونة القلب (♡) على أي عنصر.
- **"الذكاء" في الميزة:** في شاشة القنوات، القنوات المفضّلة تُرتَّب دائماً
  في أعلى القائمة تلقائياً (`sortedChannels` في `channels_screen.dart`)،
  فلا تحتاج للبحث عنها بين عشرات القنوات كل مرة.
- شاشة **المفضلة** المستقلة (تصل إليها من القائمة الجانبية) تعرض كل
  المفضلة في تبويبين: قنوات وأخبار.

## 7. هيكل الكود الكامل بعد التحديث

```
lib/
├── main.dart                       ← يُهيّئ Providers ثم يفتح شاشة الترحيب
├── config/
│   ├── api_config.dart             ← رابط Render الثابت
│   └── app_theme.dart              ← ثيمان (فاتح/مظلم) + ألوان نصوص متكيفة
├── l10n/
│   ├── app_strings.dart            ← كل النصوص (عربي/فرنسي)
│   └── app_localizations.dart      ← context.tr('key')
├── providers/
│   ├── locale_provider.dart        ← حالة اللغة الحالية
│   └── theme_provider.dart         ← حالة الوضع المظلم/الفاتح
├── models/
│   ├── channel.dart
│   └── news_article.dart
├── services/
│   ├── api_service.dart            ← الاتصال بـ Render (مهلة 60 ثانية)
│   ├── server_settings.dart        ← يُرجع رابط Render الثابت فقط
│   ├── notification_service.dart   ← إشعارات مُصلَحة (صلاحيات + قناة)
│   └── favorites_service.dart      ← المفضلة (جديد)
├── widgets/
│   └── app_drawer.dart             ← + رابط المفضلة الجديد
└── screens/
    ├── splash_screen.dart
    ├── home_screen.dart
    ├── channels_screen.dart        ← + زر المفضلة + ترتيب ذكي
    ├── news_screen.dart            ← + زر المفضلة
    ├── favorites_screen.dart       ← شاشة جديدة
    ├── settings_screen.dart        ← لغة + وضع مظلم + اختبار إشعار
    └── contact_screen.dart
```

## 8. الخطوات التالية المقترحة

1. `flutter pub get` ثم شغّل التطبيق وجرّب كل ميزة من القائمة الجانبية.
2. تأكد من ظهور الإشعار التجريبي؛ إن لم يظهر، تحقق من إعدادات إشعارات
   التطبيق يدوياً من إعدادات الهاتف (قد يكون رفضها المستخدم سابقاً قبل
   هذا الإصلاح، فيحتاج تفعيلها يدوياً مرة واحدة).
3. أضف مزيداً من النصوص المترجمة إذا وسّعت شاشات التطبيق مستقبلاً.
4. لاحقاً: مشغّل فيديو داخلي حقيقي، وربط الإشعارات بـ Firebase.
